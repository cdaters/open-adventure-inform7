# Metadata Lineage Audit

Date: 2026-06-17

This audit traces title, author, headline, and release metadata after the
Author Edition decoupling work.

## Before

`OpenAdventure.ni` had been edited to:

```inform7
"Adventure" by Craig Daters.
The story headline is "The original cave adventure game. Based on Open Adventure by Eric S. Raymond and contributors."
```

but generated outputs still contained older metadata:

```inform7
"Adventure" by Will Crowther and Don Woods.
The story headline is "The original cave adventure game".
```

The stale values appeared in:

- `OpenAdventure.inform/Source/OpenAdventure.generated.ni`
- `OpenAdventure-AuthorEdition.inform/Source/story.ni`
- generated `Release.blurb`
- generated `Metadata.iFiction`

The cause was stale generated output. `build.sh --generate-only` regenerated
`generated/*.ni` but did not compose
`OpenAdventure.inform/Source/OpenAdventure.generated.ni`; composition happened
only during compile. Author Edition release metadata was also stale until the
Author Edition was exported and translated again.

## Pipeline

Canonical metadata now originates in:

```text
OpenAdventure.ni
```

The first two source lines are the single source of truth for:

- title and author;
- story headline.

Command-line project source is generated by:

```text
build.sh
```

`build.sh` strips `Include ...` lines from `OpenAdventure.ni`, appends
generated world files and runtime modules, and writes:

```text
OpenAdventure.inform/Source/OpenAdventure.generated.ni
```

Author Edition source is generated by:

```text
tools/sync_author_edition.py
tools/make_author_edition.py
```

The exporter reads the same `OpenAdventure.ni` preamble, strips `Include ...`
lines, adds an Author Edition module spine, and writes:

```text
OpenAdventure-AuthorEdition.inform/Source/story.ni
```

Inform 7 translation then generates release metadata:

```text
OpenAdventure-AuthorEdition.inform/Release.blurb
OpenAdventure-AuthorEdition.inform/Metadata.iFiction
```

`cBlorb` embeds `Metadata.iFiction` in the generated Blorb package. In the
command-line validation run, the inspectable package was:

```text
OpenAdventure-AuthorEdition.inform/Build/output.zblorb
```

Spatterlight reads story metadata from the story file/package metadata path.
For the packaged release, the relevant source is the iFiction metadata embedded
by `cBlorb` from `Metadata.iFiction`.

## Fix

`build.sh --generate-only` now also composes
`OpenAdventure.inform/Source/OpenAdventure.generated.ni`. This makes a source
regeneration update metadata even when Inform compilation is not run.

`tests/smoke/14-metadata-lineage.sh` checks that:

- `OpenAdventure.generated.ni` matches `OpenAdventure.ni` for title/author and
  headline;
- the Author Edition exporter derives title/author and headline from
  `OpenAdventure.ni`.

## After

After regeneration and release translation, these outputs agree with
`OpenAdventure.ni`:

- `OpenAdventure.inform/Source/OpenAdventure.generated.ni`
- `OpenAdventure-AuthorEdition.inform/Source/story.ni`
- `OpenAdventure-AuthorEdition.inform/Release.blurb`
- `OpenAdventure-AuthorEdition.inform/Metadata.iFiction`
- packaged Blorb iFiction strings

Authoritative metadata source:

```text
OpenAdventure.ni
```
